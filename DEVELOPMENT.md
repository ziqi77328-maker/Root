# Root 开发原则

Root 的开发不是创作设定，而是维护发现。

任何新增内容进入仓库前，必须先通过 [validator.md](validator.md)。

## 基本原则

1. 新设定必须属于森林、方向或“我”。
2. 新设定必须至少让两个已有设定变得更合理。
3. 新设定不允许推翻 Root v1.0。
4. 不允许因为剧情需要而增加设定。
5. 不允许使用万能解释。
6. 不允许出现神、预言、命运、系统、平行宇宙等设定。
7. 世界规则必须像自然规律。
8. 规则必须违反直觉，但理解之后让人觉得本来就应该如此。
9. 如果森林会拒绝这个设定，必须直接说明原因，不能迎合。

## 新设定进入流程

1. 记录原始想法到 [notes/ideas.md](notes/ideas.md)。
2. 使用 [validator.md](validator.md) 检查。
3. 如果被拒绝，保留拒绝原因，不进入正式世界观。
4. 如果通过，写入对应文档。
5. 在 [notes/discoveries.md](notes/discoveries.md) 添加发现记录。
6. 建立与至少两个已有设定的引用关系。

## 三个预检问题

每新增一条设定，先问：

1. 它是已经确定的事实吗？
2. 它是世界的规则，还是故事里的事件？
3. 如果删掉它，Root 还能成立吗？

如果删掉它 Root 仍然成立，它暂时不属于世界观，只能留在想法或未来发现中。

## 优先级

当旧文本、灵感笔记或口头表达与根法则出现张力时，优先级如下：

1. Root v1.0 根法则
2. 已通过 Validator 的正式文档
3. 已确认发现记录
4. 未确认想法
5. 表述性、诗性或早期草稿文字
