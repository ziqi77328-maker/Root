# Root Validator

以后每提出一个设定，先检查，不先整理。

## 判断问题

### 1. 归属

它最终属于哪一个？

- 森林
- 方向
- “我”

如果答案不是三者之一，森林拒绝。

### 2. 是否违反 Root v1.0

检查它是否违反：

- 森林只有一个。
- 方向因“我”而存在。
- 每一个方向都会拥有一个使用者。
- 每一个方向都是完整的一片森林。
- 不同方向始终属于同一片森林。
- 所有方向共享森林时间。
- Root 中不存在第四种最终归属。

违反任一项，森林拒绝。

### 3. 是否解释至少两个已有设定

通过的设定必须至少让两个已有设定变得更合理。

记录格式：

- 解释了：
- 同时也解释了：

如果只服务于单一剧情、单一物象或单一场面，暂不进入正式世界观。

### 4. 是否像自然规律

它必须像自然现象一样存在。

允许违反直觉，但不允许像补丁、魔法、命令、系统功能或作者解释。

### 5. 森林会允许吗

如果它需要神、预言、命运、系统、平行宇宙、特殊选中者或万能机制才能成立，森林拒绝。

## 输出格式

### 通过

```md
结论：允许进入 Root。

归属：森林 / 方向 / “我”

解释的旧设定：
- ...
- ...

自然规律表述：
...

应加入文档：
- ...
```

### 拒绝

```md
结论：森林拒绝。

原因：
- ...

冲突规则：
- ...

可保留位置：
- notes/ideas.md
```
